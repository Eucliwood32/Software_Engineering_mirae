# Requirements Traceability Matrix

## 1. Purpose

This matrix records the current SRS-to-design-to-test traceability state for HISYS EduOps. It is a controlled draft baseline for implementation and TDD planning, not a claim that every requirement already has executable tests.

## 2. Current readiness conclusion

The SRS has stable requirement IDs and acceptance seeds, but traceability is not yet sufficient for full TDD implementation because many SRS rows are covered only by grouped fixture sections rather than exact requirement-to-test mappings. This matrix shall be maintained until every implemented behavior has a requirement anchor, design/interface anchor, STD or fixture gate, and RED/GREEN test evidence.

## 3. Matrix status model

| Status | Meaning |
|---|---|
| Exact | Requirement ID is explicitly referenced in STD/VVP/design mapping. |
| Grouped | Requirement is covered by a scenario/test group, but the exact ID is not yet mapped. |
| Gap | No credible test/design mapping found. |
