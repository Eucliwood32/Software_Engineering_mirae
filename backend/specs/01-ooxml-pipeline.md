# Spec 01: OOXML 문서 분석 파이프라인

> **대응 요구사항:** FR-1.1, FR-1.2
> **담당:** 백엔드(조원희) + 비즈니스 로직(김휘중)

---

## FR-1.1 — .pptx / .docx 파일 파싱 및 글자 수 추출

### 가치
조장이 문서 기여량을 자동 집계하여 수작업 없이 팀원별 작성 분량을 파악할 수 있다.

### 구현 기준 (How)

| 담당 | 구현 내용 |
|---|---|
| **백엔드** | `python-pptx`, `python-docx` 라이브러리로 파일을 열고 XML 구조를 순회하여 텍스트·도형 원시 데이터를 추출한다. |
| **프론트엔드** | 분석할 파일을 선택하는 파일 선택 다이얼로그 UI를 제공하고, 파싱 진행 상태(Progress Bar)를 표시한다. |
| **비즈니스 로직** | 추출된 원시 텍스트에서 공백·개행을 제거한 순수 글자 수를 계산하고, 도형 개수를 카운트하는 집계 로직을 구현한다. |

#### 세부 사항
- `.pptx`: 각 슬라이드(`prs.slides`)의 모든 `shape.text_frame.text`를 순회하여 텍스트 추출
- `.docx`: `document.paragraphs`를 순회하여 `paragraph.text` 추출
- 글자 수 계산: `len(text.replace(' ', '').replace('\n', ''))` — 공백·개행 제외
- 도형(shape) 개수도 별도 카운트

### 수용 기준 (Acceptance Criteria)

| # | 기준 | 판정 방법 |
|---|---|---|
| AC-1 | 텍스트 200자가 포함된 테스트용 `.docx` 파일 → 출력 글자 수 `200 ± 0` | 정확히 일치 |
| AC-2 | 슬라이드 3장, 각 슬라이드에 텍스트 박스 2개(각 50자) `.pptx` → 총 글자 수 `300`, 도형 수 `6` | 정확히 일치 |
| AC-3 | 빈 파일(내용 없음) → 글자 수 `0`, 도형 수 `0`, 예외 미발생 | 정상 반환 |

## FR-1.2 — 팀원 식별자 기반 기여 데이터 재정렬

### 가치
파일 단위가 아닌 팀원 단위로 데이터를 집계하여 "누가 얼마나 썼는지" 즉시 확인할 수 있다.

### 구현 기준 (How)

| 담당 | 구현 내용 |
|---|---|
| **백엔드** | `.docx`의 `core_properties.author`, `.pptx`의 `core_properties.last_modified_by` 메타데이터 필드를 읽어 팀원 식별자를 추출한다. |
| **비즈니스 로직** | 파싱 결과를 `Dict[str, int]` 구조로 팀원별 합산 집계하고, 식별자가 없는 경우 `"Unknown"` 키로 분류하는 로직을 구현한다. |

#### 세부 사항
- 집계 구조: `Dict[str, int]` (`{"팀원이름": 총_글자수}`)
- `.docx` 작성자: `core_properties.author`
- `.pptx` 작성자: `core_properties.last_modified_by`
- 식별자 비어있음(`""` 또는 `None`): `"Unknown"` 키로 분류

### 수용 기준 (Acceptance Criteria)

| # | 기준 | 판정 방법 |
|---|---|---|
| AC-1 | `"Alice"` 100자 + `"Bob"` 150자 `.docx` → `{"Alice": 100, "Bob": 150}` | 딕셔너리 일치 |
| AC-2 | 작성자 메타데이터 없는 파일 → `"Unknown"` 키 존재, 예외 미발생 | 정상 반환 |
