# Spec 05: 결과 시각화 및 대시보드

> **대응 요구사항:** FR-5.1, FR-5.2, FR-5.3
> **담당:** 프론트엔드(이대한) + 비즈니스 로직(김휘중) + 백엔드(조원희)

---

## FR-5.1 — 팀원별 기여도 차트 렌더링

### 가치
조장이 숫자 데이터가 아닌 시각적 그래프로 팀원 간 기여 현황을 한눈에 비교할 수 있다.

### 구현 기준 (How)

| 담당 | 구현 내용 |
|---|---|
| **프론트엔드** | `matplotlib`로 막대 차트·레이더 차트를 렌더링하고 UI 탭/패널에 삽입한다. 초기 상태 안내 텍스트 표시를 제어한다. |
| **비즈니스 로직** | 최종 기여도 매트릭스를 차트 입력 형식(`팀원명: [git, doc, msg, total]`)으로 변환하는 데이터 포맷팅 함수를 구현한다. |

#### 세부 사항

**종합 기여도 막대 차트:**
- X축: 팀원명
- Y축: 최종 기여도 점수 (0.0~1.0)

**지표별 레이더(방사형) 차트:**
- 각 팀원의 Git·OOXML·메신저 세 지표 정규화 점수를 꼭짓점으로 표시

**초기 상태:**
- 분석 미실행 시: `"분석을 실행하면 결과가 표시됩니다."` 텍스트

### 수용 기준 (Acceptance Criteria)

| # | 기준 | 판정 방법 |
|---|---|---|
| AC-1 | 팀원 4명 분석 → 막대 차트 4개 막대 + 팀원 이름 레이블 | 시각적 확인 |
| AC-2 | 레이더 차트 꼭짓점 레이블: `"Git"`, `"문서"`, `"메신저"` 3개 표시 | 레이블 확인 |
| AC-3 | 분석 미실행 → `"분석을 실행하면 결과가 표시됩니다."` 표시 | 텍스트 확인 |

## FR-5.2 — 최종 리포트 로컬 파일 저장 (.md / .csv)

### 가치
조장이 분석 결과를 팀 공유용 문서로 즉시 내보낼 수 있다.

### 구현 기준 (How)

| 담당 | 구현 내용 |
|---|---|
| **백엔드** | `.csv`는 `utf-8-sig`, `.md`는 `utf-8` 인코딩으로 원자적 저장한다. |
| **프론트엔드** | `[리포트 저장]` 버튼, 파일 저장 다이얼로그, 저장 완료 후 상태바 3초 표시를 구현한다. |
| **비즈니스 로직** | 기여도 매트릭스를 마크다운 테이블 문자열 또는 CSV 행 문자열로 직렬화하는 포맷팅 함수를 구현한다. |

#### 세부 사항

**.md 출력 형식:**
```markdown
# QCE 기여도 분석 리포트
**프로젝트:** [프로젝트명]  
**분석 일시:** [YYYY-MM-DD HH:MM]

| 이름 | Git점수 | 문서점수 | 메신저점수 | 최종점수 |
|---|---|---|---|---|
```

**.csv 출력 형식:**
```
author,git_score,doc_score,msg_score,total_score
```
- 인코딩: UTF-8 BOM (`utf-8-sig`) — Excel 한글 호환

**저장 성공:** 상태바에 파일 경로 포함 텍스트 3초간 표시

### 수용 기준 (Acceptance Criteria)

| # | 기준 | 판정 방법 |
|---|---|---|
| AC-1 | `.md` 저장 후 → 마크다운 테이블 올바른 형식 렌더링 | 텍스트 검증 |
| AC-2 | `.csv` 저장 후 Excel 열기 → 한글 팀원명 깨짐 없음 | 인코딩 확인 |
| AC-3 | 저장 완료 → 상태바 경로 텍스트 3초 이상 표시 후 사라짐 | UI 확인 |

## FR-5.3 — 결측 데이터 경고 인터페이스

### 가치
조장이 리포트에 누락된 데이터가 있음을 인지하고 평가 결과를 맥락과 함께 해석할 수 있다.

### 구현 기준 (How)

| 담당 | 구현 내용 |
|---|---|
| **프론트엔드** | 메신저 결측 플래그 `True` → 노란 배경 경고 배너 표시, `False` → 숨김 |
| **백엔드** | 리포트 저장 시 결측 플래그 `True` → `.md`/`.csv` 파일 하단에 경고 텍스트 삽입 |
| **비즈니스 로직** | 오케스트레이터에서 `msg_missing: bool` 플래그를 집계하여 전달 |

#### 경고 텍스트

**UI 대시보드:** 노란색 배경 배너
```
⚠ 메신저 데이터 형식 불일치로 인해 해당 지표가 평가에서 제외되었습니다.
```

**.md 리포트:**
```markdown
> ⚠ **경고:** 메신저 데이터 형식 불일치로 인해 해당 지표가 평가에서 제외되었습니다.
```

**.csv 리포트:** (빈 행 후)
```
"WARNING","메신저 데이터 형식 불일치로 인해 해당 지표가 평가에서 제외되었습니다."
```

### 수용 기준 (Acceptance Criteria)

| # | 기준 | 판정 방법 |
|---|---|---|
| AC-1 | 메신저 미입력 분석 → 노란 배경 경고 배너 표시 | UI 확인 |
| AC-2 | 동 조건 `.md` 저장 → 경고 블록쿼트 포함 | 텍스트 확인 |
| AC-3 | 메신저 정상 입력 → 경고 배너 + 텍스트 미표시 | 부재 확인 |
